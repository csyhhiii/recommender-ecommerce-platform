# 6 系统测试

## 6.1 测试意义及目的

系统测试是电子商务推荐平台上线前的关键质量保障环节，其意义在于通过“功能正确性 + 业务流程完整性 + 关键异常可控性”的综合验证，确认系统能够在真实业务场景下稳定运行。结合本项目（Spring Boot 后端 + Vue 前端 + MySQL 数据层 + 推荐服务模块）的实现，系统测试的主要目的如下：

1. **验证核心业务闭环是否可用**：确保“用户注册/登录 → 商品浏览与检索 → 购物车 → 下单支付（模拟）→ 订单查询/物流跟踪 → 评价反馈”链路贯通。
2. **验证推荐功能是否有效集成**：确认推荐中心能够基于用户行为、收藏、订单等数据返回个性化商品结果，且推荐接口可正常调用。
3. **验证多角色权限与菜单控制**：验证普通用户与后台管理员在登录后获得不同菜单、页面与接口权限，避免越权访问。
4. **验证异常输入与边界条件处理**：包括登录失败提示、验证码校验失败、参数校验不通过、未登录访问受限页面拦截等。
5. **验证系统可维护性与可扩展性基础**：通过对用户管理、商品管理、订单管理、拼团、客服、字典管理等模块的联合测试，确认模块化设计满足后续迭代需要。

综上，系统测试不仅用于发现缺陷，更用于为系统交付提供可量化、可追溯的质量依据。

## 6.3 测试过程

为保证测试覆盖典型业务场景，本节按照“前台用户操作 + 后台管理操作 + 推荐能力验证 + 异常场景验证”展开说明。

### （1）系统登录测试

用户登录系统时，需输入正确的账户名和密码，并填写验证码；系统前端会先进行表单校验（用户名、密码长度），随后向后端登录接口发送请求。若账户名、密码或验证码有误，系统弹出对应错误提示；若正确，则按角色进入前台首页或后台首页。

- **测试步骤**：
  1. 打开登录页，输入错误用户名或密码并提交。
  2. 输入正确用户名密码，但输入错误验证码并提交。
  3. 输入全部正确的信息并提交。
- **预期结果**：
  - 错误账户/密码：提示登录失败信息；
  - 错误验证码：提示“验证码错误”；
  - 正确登录：弹出“登录成功”，并根据角色跳转至对应页面。
- **实际结果**：与预期一致，测试通过。

### （5）后台管理与权限控制测试

后台管理员登录后，系统根据动态菜单加载管理页，验证用户管理、商品管理、订单管理、库存管理、拼团管理等页面可访问；普通用户不具备后台菜单访问权限。

- **测试步骤**：
  1. 使用管理员账号登录后台；
  2. 依次进入多个管理模块执行查询/新增/编辑操作；
  3. 使用普通用户账号尝试访问后台路由。
- **预期结果**：
  - 管理员可正常访问后台模块；
  - 普通用户被限制访问后台受控资源。
- **实际结果**：测试通过。
