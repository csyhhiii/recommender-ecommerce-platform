// 需要安装: npm install babel-plugin-transform-remove-console --save-dev

      config.plugin('remove-console')
        .use(require('babel-plugin-transform-remove-console'), [{
          exclude: ['error', 'warn'] // 保留 error 和 warn
        }])
    }

## 🎯 三、推荐系统前台化改造

### 改造目标

将推荐算法从"看不见的后台"转变为"可感知、可交互、可解释"的前台功能，让用户能够：
- **可感知**：看到推荐算法的运行状态和统计信息
- **可交互**：对推荐结果进行反馈，影响后续推荐
- **可解释**：了解为什么推荐这些商品

### 实现内容

#### 1. 后端改造

##### 1.1 新增 / 扩展 DTO 类

- **RecommendationExplanation.java**
  - 推荐解释信息（已扩展为多算法可解释结构），包含：
    - `productId`: 推荐商品ID
    - `score`: 综合推荐分数
    - `reasonType`: 推荐理由类型（`COLLABORATIVE` / `POPULAR` / `SIMILAR_USER` / `HYBRID` 等）
    - `reasonDescription`: 推荐理由描述（面向用户的完整文案）
    - `similarUserCount`: 相似用户数量
    - `similarUserHints`: 相似用户提示（脱敏，如"用户1(相似度 92%)"）
    - `sourceProductIds`: 来源商品ID列表（与用户历史行为相关）
    - `confidence`: 算法置信度（0-1）
    - `timestamp`: 推荐时间戳
    - `primaryReason`: 主推荐理由摘要（用于前端优先展示）
    - `algorithmScores: Map<String, Double>`：多算法得分
    - `supportingFactors: List<SupportingFactor>`：支撑因素列表

- **RecommendationResponse.java**
  - 推荐响应，包含：
    - `products`: 推荐商品列表
    - `explanations`: 推荐解释列表（与 `products` 一一对应）
    - `algorithmType`: 算法类型
    - `stats`: 统计信息（相似用户数、行为数、执行时间、是否降级）

##### 1.2 新增实体类

- **RecommendationFeedback.java**
  - 推荐反馈实体，记录用户对推荐商品的反馈
  - 字段：`userId`, `productId`, `feedbackType`, `feedbackTime`, `remark`
  - `feedbackType` 取值（统一大写）：
    - `LIKE`：喜欢
    - `DISLIKE`：不喜欢
    - `ALREADY_OWN`：已拥有
    - `NOT_INTERESTED`：不感兴趣
    - `TOO_EXPENSIVE`：太贵
    - `WILL_BUY_LATER`：稍后购买

##### 1.3 反馈权重与降权策略

- 内部配置常量 `FEEDBACK_WEIGHTS: Map<String, Double>`：
  - `LIKE` → `1.1`（轻微加权）
  - `DISLIKE` → `0.1`（降权 90%）
  - `ALREADY_OWN` → `0.0`（直接过滤，不再推荐）
  - `NOT_INTERESTED` → `0.3`（大幅降权）
  - `TOO_EXPENSIVE` → `0.5`（中等降权）
  - `WILL_BUY_LATER` → `0.7`（轻降权）

##### 1.4 控制器扩展

- **GET /recommend/user/{userId}/explanation** - 获取带解释的推荐
- **POST /recommend/feedback** - 保存用户反馈

#### 2. 前端改造

##### 2.1 新增页面

- **RecommendCenter.vue** - 推荐中心页面
  - **可感知功能**：展示推荐统计卡片
  - **可解释功能**：折叠面板展示详细解释、算法得分雷达图、支撑因素时间线
  - **可交互功能**：喜欢/不喜欢按钮、更多反馈下拉菜单

##### 2.2 首页改造

- 在"为您推荐"区域添加"推荐中心"入口按钮
- 仅登录用户可见

##### 2.3 路由配置

- 新增路由：`/recommend-center`
- 需要登录认证

#### 3. 数据库改造

- **recommendation_feedback** 表
  - 存储用户对推荐商品的反馈
  - 索引：`user_id`, `product_id`
  - 外键：关联`user`和`product`表

### 使用说明

1. **数据库初始化**：执行SQL文件中的`recommendation_feedback`表创建语句
2. **访问推荐中心**：用户登录后，在首页"为您推荐"区域点击"推荐中心"按钮
3. **查看推荐解释**：点击每个推荐商品下方的"为什么推荐这个商品？"折叠面板
4. **反馈推荐结果**：使用喜欢/不喜欢按钮或更多反馈菜单进行反馈

### 技术亮点

1. **推荐解释生成**：算法在计算推荐时同步生成多维度解释信息
2. **多算法融合解释**：通过 `algorithmScores` 暴露多算法得分
3. **用户反馈精细化**：支持多种反馈类型，通过权重和屏蔽策略精细影响推荐
4. **可视化展示**：使用进度条、标签、雷达图、时间线等 UI 组件
5. **实时交互闭环**：用户反馈 → 服务端记录并更新权重 → 下次推荐即时可见效果

### 后续优化方向

1. **反馈权重优化**：根据反馈时间、反馈频率动态调整权重
2. **多算法融合**：结合内容推荐、热门推荐等多种算法
3. **A/B测试**：测试不同解释方式对用户满意度的影响
4. **推荐效果分析**：统计反馈数据，分析推荐质量
